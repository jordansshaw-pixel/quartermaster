# Client Brief — The Quartermaster

**Client:** me — a solo operator running personal, client, and founder-venture
projects across several companies, built largely with AI.

**What's broken.** I build faster than I track and have lost the thread of what's
running — scattered across machines, domains, DNS, registrars, SaaS accounts, and
repos. Some exist only as an API key in a config; some log in under a personal
email tied to a business. A portal I built has logins now lost. Charges hide
across inboxes, app stores, and cards. Assets blur across businesses with no map
of what's whose — and it compounds as I build.

**What I've tried.** Every fix rots the same way: nothing forces verification, so
stale entries look live, and they confuse the account that *controls* an asset
with the business that *owns* it.

**What I need.** One specialist that turns this into a single source of truth I
answer once and it remembers — not re-answer or guess.

**Desired outcome.** For any project or asset, four answers I can trust: its lane
(personal / client / which business) and status; the accounts, sites, logins and
emails it runs on; what I pay and when the next charge hits; and which login
controls it, distinct from which business owns it.

And it must stay true. *Self-maintaining* means two things: anything unconfirmed
**surfaces itself** instead of pretending it works, and a defined reconfirmation
**cadence** keeps verification happening — because the rot is a cadence gap, not
a storage gap.
